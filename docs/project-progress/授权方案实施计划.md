## 实施步骤

### Step 1: 机器指纹 ✓ (已完成)

**目标:** 生成稳定的硬件指纹，用于授权绑定。

**产出文件:**

```
tools/tiaportal-mcp/src/TiaMcpServer/License/MachineId.cs
```

**方案:**

- WMI 采集: `Win32_Processor.ProcessorId` + `Win32_BaseBoard.SerialNumber` + 主网卡 MAC
- SHA256 哈希 → 前 16 字符 → `XXXX-XXXX-XXXX-XXXX` 格式
- 降级策略: 缺一项时前一项加倍权重；三项全失败才抛异常
- 过滤: 跳过全零、`To be filled by O.E.M.`、虚拟网卡

**验证:**

```bash
TiaMcpServer.exe --show-machine-id
# 输出: 2A99-469A-B303-AE1D
```

**注意要点:**

- 物理机上测试，虚拟机 MAC 是 VMware 固定值
- 换 CPU/主板会导致指纹变化 → 需重新激活（合理行为）
- WMI 查询可能被安全软件拦截，需有降级策略

### Step 3: 本地离线授权模式

**目标:** 实现不依赖网络的授权校验。使用 RSA 签名防止 license 文件被伪造。

**产出文件:**

```
tools/tiaportal-mcp/src/TiaMcpServer/License/
├── MachineId.cs           ← 已有
├── LicenseCache.cs        ← 新增: 本地加密 token 读写
└── LicenseValidator.cs    ← 新增: 校验入口（在线+离线）
```

#### 3.1 LicenseCache.cs — 本地缓存

**方案:**

- 存储位置: `%ProgramData%\TiaMcpServer\.license_cache`
- 格式: 文件内容 AES-256-GCM 加密
- 解密后是 JWT:
  ```json
  {
    "machineId": "2A99-469A-B303-AE1D",
    "exp": 1750000000
  }
  ```
- 验证时:
  1. 用内置 RSA 公钥验 JWT 签名 → 防篡改
  2. 比对 machineId → 防复制到别的机器
  3. 检查 exp → 防过期
  4. 三项全通过 → 本地有效，不联网

**注意要点:**

- RSA 公钥硬编码在 exe 里。攻击面是反编译 exe 换公钥 → Phase 1 先不做混淆，Phase 7 加壳
- AES 密钥从 MachineId 派生（只有本机能解密自己的缓存）
- 缓存过期时间 30 天，和网络续期间隔一致

#### 3.2 LicenseValidator.cs — 校验入口

**方案:**

```
Validate(key) 流程:
  1. 存在 --license-key?
     ├─ 否 → 退出
     └─ 是 → 2
  2. LicenseCache.TryValidate() → 本地有效?
     ├─ 是 → 启动
     └─ 否 → 3
  3. 在线校验 → POST /api/validate
     ├─ 成功 → LicenseCache.Save(token) → 启动
     └─ 失败 → 输出错误原因 → 退出
```

**注意要点:**

- 超时 10 秒（工厂网络可能慢但不能无限等）
- 错误信息要友好: "授权已过期 (2027-01-01)" 而非 "HTTP 500"
- 不要在日志里打印完整 Key，只打前缀 `TIA-****`

#### 3.3 Program.cs 改动

```csharp
// 在 Openness.Initialize() 之后, RunStdioHost/RunHttpHost 之前插入:

var licenseOk = await LicenseValidator.Validate(options.LicenseKey);
if (!licenseOk)
{
    LogDiag("LICENSE DENIED");
    Environment.Exit(7);
}
LogDiag("LICENSE OK");
```

**注意要点:**

- 改动量: Program.cs 加 ~15 行 + 一个 using
- 不影响 RunStdioHost 和 RunHttpHost 的任何逻辑
- `Environment.Exit(7)` 退出码，方便客户排查

#### 3.4 密钥对生成（离线第一步）

在进入 Step 4 之前，先生成 RSA 密钥对:

```bash
openssl genrsa -out license_private.pem 2048
openssl rsa -in license_private.pem -pubout -out license_public.pem
```

- 私钥 (`license_private.pem`): 你自己保管，用于签发 license token
- 公钥 (`license_public.pem`): 硬编码到 `LicenseValidator.cs` 中

**注意要点:**

- 私钥绝对不能泄露、不能提交到 Git
- 公钥可以公开，硬编码在源码里没问题
- 建议私钥离线存储（加密 U 盘），只在签发时拿出来

### Step 4: 在线授权服务器（含云部署学习）

**目标:** Flask 授权服务器 + 部署到阿里云。

**产出文件 (独立仓库，不在 TIA MCP 仓库里):**

```
license-server/
├── app.py              ← Flask 主程序
├── models.py           ← SQLite 数据模型
├── admin.html          ← 简易管理界面
├── keygen.py           ← Key 生成工具 (你本地用)
├── requirements.txt
└── deploy/
    ├── nginx.conf      ← Nginx 反向代理配置
    └── mcp-license.service  ← systemd 服务
```

#### 4.1 数据库设计

```sql
-- SQLite 单文件
CREATE TABLE license_keys (
    id            INTEGER PRIMARY KEY,
    key_hash      TEXT NOT NULL UNIQUE,   -- bcrypt(Key)
    key_prefix    TEXT NOT NULL,          -- "TIA-"
    max_machines  INTEGER DEFAULT 1,      -- 允许绑定的机器数
    issued_to     TEXT,                   -- 客户名/公司
    issued_at     TEXT NOT NULL,          -- ISO 8601
    expires_at    TEXT NOT NULL,          -- ISO 8601
    revoked       INTEGER DEFAULT 0
);

CREATE TABLE machines (
    id            INTEGER PRIMARY KEY,
    key_id        INTEGER REFERENCES license_keys(id),
    machine_id    TEXT NOT NULL,           -- 硬件指纹
    first_seen    TEXT NOT NULL,
    last_seen     TEXT NOT NULL,
    UNIQUE(key_id, machine_id)
);
```

#### 4.2 API 设计

| 端点 | 方法 | 功能 | 说明 |
|------|------|------|------|
| `/api/activate` | POST | 首次激活 | key + machineId → 绑机器 |
| `/api/validate` | POST | 续期/校验 | key + machineId → 新 token |
| `/health` | GET | 健康检查 | 无需认证 |

**激活流程 (active):**

```
请求: { "key": "TIA-A7X9-...", "machineId": "2A99-..." }

服务端:
  1. bcrypt.verify(key, stored_hash) → 找到 Key 记录
  2. 检查: 未过期? 未吊销? 机器数未超限?
  3. 如果是新机器: 记录到 machines 表
  4. 生成 JWT: { machineId, exp: now+30d }
  5. RSA 私钥签名
  6. 返回: { "token": "eyJ...", "expiresAt": "2026-07-20" }

客户端:
  收到 token → LicenseCache.Save(token)
  → 30 天内不再联网
```

#### 4.3 授权服务器技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3.11+ | 你熟悉 |
| 框架 | Flask + Gunicorn | 简单够用 |
| 数据库 | SQLite | 单文件，请求量低 |
| 反向代理 | Nginx | HTTPS 终止 |
| 部署 | 阿里云 ECS 2核2G Ubuntu 22.04 | 最便宜 |

#### 4.4 云部署步骤

```
1. 买阿里云 ECS (2核2G, Ubuntu 22.04)
2. 安全组开放: 22(SSH) + 80(HTTP) + 443(HTTPS)
3. SSH 上去:
   apt update && apt install python3-venv nginx
4. 创建 venv + pip install flask gunicorn bcrypt pyjwt cryptography
5. 上传 license-server/ 代码
6. 配置 Nginx 反向代理 + Let's Encrypt HTTPS
7. systemd 启动 Gunicorn
8. 测试: curl https://license.yourdomain.com/health
```

**注意要点:**

- 数据库文件 (`license.db`) 定期备份（crontab 每天一次）
- Nginx 配置限流（防止暴力破解 API Key）
- `/admin` 页面加 HTTP Basic Auth 或 VPN 访问
- 域名建议用二级域名如 `license.tia-mcp.com`

#### 4.5 keygen.py — Key 管理工具

```bash
# 生成新 Key
python keygen.py create --tier pro --issued-to "某公司" --expires 2027-06-01
# → TIA-A7X9-B3KF-M2QN-W5PR

# 查看 Key 状态
python keygen.py info TIA-A7X9-B3KF-M2QN-W5PR
# → tier: pro, machines: 1/3, expires: 2027-06-01

# 吊销 Key
python keygen.py revoke TIA-A7X9-B3KF-M2QN-W5PR

# 列出所有 Key
python keygen.py list
```

### Step 5: 编译 + 加壳 + 发布

**目标:** 产出闭源的 exe，增加反编译门槛。

#### 5.1 编译

```bash
# V20 (TIA Portal V20, .NET Framework 4.8)
dotnet build tools/tiaportal-mcp/src/TiaMcpServer/TiaMcpServer.csproj -c Release

# V21 (TIA Portal V21, .NET 8)
dotnet build tools/tiaportal-mcp/src/TiaMcpServer/TiaMcpServer.V20.csproj -c Release
```

**注意要点:**

- 编译必须在安装了对应版本 TIA Portal 的 Windows 机器上
- Release 模式去除调试符号
- 确认 `AssemblyVersion` 改为新版本号（如 `2.3.0`）

#### 5.2 加壳

**推荐工具:** ConfuserEx (开源) 或 .NET Reactor (付费)

```bash
ConfuserEx.CLI.exe -n TiaMcpServer.crproj
```

**混淆选项:**

- 符号重命名 (rename): 类名/方法名 → 随机字符串
- 控制流混淆 (ctrl flow): 打乱 IL 指令顺序
- 反调试 (anti debug): 检测调试器附加
- **重点保护:** `LicenseValidator.cs` 中的公钥和校验逻辑

**注意要点:**

- .NET Reactor (~$200) 比 ConfuserEx 效果好很多
- 混淆后务必测试: 有些混淆选项会破坏 MCP SDK 的反射调用
- `[McpServerTool]` 标记的方法不能重命名（反射靠 Attribute 发现，但方法名也参与 MCP 协议）

## 风险清单

| 风险 | 影响 | 缓解 |
|------|------|------|
| WMI 被安全软件拦截 | 指纹获取失败 | 降级策略（三项任取二） |
| 工厂网络断外网超 30 天 | 无法续期 | 提供离线激活码备选方案 |
| 反编译绕过授权 | 盗版 | 加壳 + 法律条款 |
| 上游仓库大重构 | 合并冲突 | 每月合并一次，不长期积压 |
| 授权服务器宕机 | 新客户无法激活 | 已激活用户 30 天缓存不受影响 |
| 私钥泄露 | 可伪造 license | 私钥离线存储，定期更换密钥对 |
